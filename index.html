Thoughts on Progressive Enhancement

Prior to the early 2000s, web development followed the philosophy of Graceful Degradation. Its focus was on building sites for the newest versions of browsers. If opened on an older browser or one that lacked the required functionality, it would “degrade” to a more basic site. The problem with this approach is that the degraded sites could have all sorts of errors. The developer, if they chose to do so, would need to test the site on a variety of browsers and versions to find and correct possible errors. In 2003, Steven Champeon and Nick Finck coined the term Progressive Enhancement when describing a new way of thinking about web development philosophy. The focus is placed on the content, not the browser. This is essentially the opposite approach from Graceful Degradation. The idea is to create valid and complete base level content, the html. Next, add a layer of style to that, the css. Then, add a layer of scripting to the top. This way the content can be enjoyed or accessed the way it was intended and in its entirety at the base level. However, If the user’s browser allows, the site can be viewed with style and scripting for the full experience. Focusing on content first has another advantage. Since search engines only look at base level content, putting more emphasis on that should yield better search results.
